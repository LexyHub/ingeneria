A continuación, se presenta la guía técnica definitiva del "Estándar Lexy", un compendio exhaustivo que integra **todos** los ejemplos visuales, conceptuales y técnicos discutidos, eliminando ambigüedades y asegurando especificaciones precisas para una implementación frontend robusta y coherente, ya sea por un desarrollador o una IA.

*(Nota para IA/v0: Esta guía asume la disponibilidad local del archivo vectorial `logolexy.svg` y los archivos de fuente `Archivo` WOFF2, que deben ser referenciados correctamente en la implementación.)*

**Guía Técnica Definitiva de Diseño Visual y CSS "Estándar Lexy" (v13 - Arquitectura Integrada)**

**0. Introducción y Visión General**

*   **Propósito:** Especificar de forma técnica y detallada el sistema de diseño visual "Lexy" para asegurar la consistencia y calidad en todas las interfaces digitales (Fichas de Usuario Web/Móvil, Dashboards SaaS, Herramientas Internas).
*   **Audiencia:** Desarrolladores Frontend (React) e Inteligencia Artificial de generación de código (ej. V0).
*   **Metodología:** Esta guía se basa en el análisis de múltiples ejemplos de interfaces existentes y define tokens de diseño (Variables CSS), tipografía, sistemas de layout responsivo, y estilos de componentes detallados, incluyendo variantes y estados interactivos.
*   **Restricción Fundamental:** Se debe utilizar **exclusivamente CSS estándar** y las variables CSS definidas en esta guía. **Está estrictamente prohibido el uso de sintaxis o clases de frameworks CSS como Tailwind (@apply, clases utilitarias compuestas).**

**1. Principios de Diseño Fundamentales ("Guía de Visión Lexy")**

*   **Interfaz Limpia:** Minimalismo, priorización de contenido, eliminación de ruido visual, uso intencional del espacio blanco.
*   **Jerarquía Visual Clara:** Guía efectiva del usuario a través de la escala tipográfica, pesos, colores y espaciado. Elementos críticos deben ser inmediatamente identificables.
*   **Consistencia Rigurosa:** Aplicación estricta y uniforme de todos los elementos del Design System (tokens, componentes, layouts) en todas las interfaces.
*   **Feedback Visual Inmediato:** Proveer respuestas claras y predecibles a las interacciones del usuario (estados :hover, :focus-visible, :active, :disabled), validaciones y acciones críticas (ej. modales, notificaciones).
*   **Funcionalidad Directa:** Diseño pragmático y eficiente, optimizado para flujos de trabajo legales y de gestión de datos, facilitando la comprensión y ejecución de tareas.
*   **Adaptabilidad Responsiva:** Experiencia de usuario fluida y optimizada en los breakpoints definidos (Móvil ≤375px, Desktop ≥1440px).

**5. Arquitectura de Página (Layouts)**

*   **`lexy-layout-ficha` (Full Page - Fichas/Estudio):**
    *   Uso: Formularios lineales, visualización detallada.
    *   Estructura Desktop: `[Header Fijo opcional: Stepper/Tabs] + ContentArea(max-width: 1112px, margin: auto, padding-lateral: 164px) + [Footer Fijo opcional: Atrás]`.
    *   Estructura Móvil: `Header Fijo App + Header Fijo ProgressBar + ContentArea(padding: 24px) Scrollable + Home Indicator`.
    *   Fondo: `var(--color-bg-app-secondary)`.
*   **`lexy-layout-saas` (Split Page - Dashboards/Herramientas):**
    *   Uso: Dashboards, gestión compleja, navegación multi-sección.
    *   Estructura Desktop: `Header Fijo (SaaS Style) + [Sidebar(width: 355px, padding: 28px) Fija + ContentArea(flex-grow:1, padding: 28px) Scrollable]`. Generalmente sin Footer de página.
    *   Fondo (Content): `var(--color-bg-app-primary)`.
*   **Grid System:**
    *   Móvil (≤375px): 4 columnas, gutter `16px`.
    *   Desktop (≥1440px): 12 columnas, gutter `24px` (Ficha) / `28px` (SaaS).

*(Asegurar que todos los componentes se definan con CSS estándar y variables)*

*   **Botones `.lexy-button`:**
    *   Base, Tamaños (`--standard`, `--large`), Variantes (`--primary-blue`, `--secondary-blue`, `--primary-green`, `--secondary-green`, `--solid-pink`, `--outline-neutral`, `--social`, `--table-add`, `--table-delete`, `--help-float`).
    *   Estados Interactivos:

        ```css
            color: var(--color-text-disabled) !important;
            border-color: var(--color-border-input) !important; /* Para variantes outline */
            outline: none !important;
        }
        /* Considerar :active para feedback al click */
        .lexy-button:active { transform: translateY(1px); /* Ejemplo */ }
        ```
*   **Inputs & Fields `.lexy-input`, `.lexy-select`, etc.:** Wrapper con borde base, focus y error. Input/Select con estilos de texto, padding y placeholder. Estado filled (`background: #d4d4d4`).
*   **Selects/Autocomplete (Dropdowns):** Lista `.lexy-select__dropdown`/`__suggestions` con scrollbar custom (thumb `primary`, track `#E6E6E6`), item (`__option`) con hover (`bg-hover-secondary`).
*   **Tablas `.lexy-table`:** Header (`--bg-table-header`), celdas con bordes (`--color-border-divider`), radios en esquinas. Responsive.
*   **Stepper `.lexy-stepper` (Desktop):** Estados visuales `active`/`completed` para círculos y líneas.
*   **Progress Bar `.lexy-mobile-progress-nav` (Mobile):** Barra, segmentos (`active`/`inactive`), texto "X de Y", botones nav.
*   **Tabs `.lexy-tabs`:**
    *   Estándar (`--standard`): Item activo con `border-bottom: 3px solid var(--color-primary);` y texto `--color-primary`.
    *   SVG (`--svg`): Fondos SVG verde/gris.
*   **Modales `.lexy-modal`:** Overlay, Contenedor. Variantes Informativo y Confirmación (título negro).
*   **Sidebar `.lexy-sidebar` (SaaS):** Títulos, Items (`--active`), Subitems, Badges.
*   **Header `.lexy-header` (SaaS):** Logo, Searchbar, Acciones (Bell), User Menu (Avatar).
*   **Cards `.lexy-card`, `.lexy-list-card`:** Contenedores con padding, radius, bg, borde/sombra. Estructura específica para `.lexy-list-card`.
*   **Tooltips `.lexy-tooltip`:** Estilo negro, posicionamiento absoluto.

**8. Responsividad**

*   Implementar `@media (max-width: 375px)` y `@media (min-width: 1440px)`.
*   Ajustar tokens de espaciado y tipografía.
*   Adaptar Grids y Layouts.
*   Utilizar componentes de navegación específicos (Stepper vs Progress Bar).
*   Asegurar funcionalidad de Tablas en móvil (Scroll H o Stacked).

9. Buenas Prácticas de Diseño y Experiencia de Usuario (UX/UI) - Directrices de Aplicación

Más allá de la replicación técnica de los estilos definidos, la implementación del Estándar Lexy debe guiarse por principios sólidos de UX/UI para crear interfaces efectivas y agradables. Considera esta sección como una capa de refinamiento sobre la base técnica:

Coherencia y Ritmo Espacial: Utiliza consistentemente las variables de espaciado (--space-*, --gutter-*) no solo dentro de los componentes, sino entre ellos y las secciones principales. Busca un flujo visual armonioso y evita tanto la congestión como el exceso de espacio vacío. El espaciado definido (16px, 24px, 36px, etc.) debe ser la norma para márgenes y gaps entre elementos discretos.

Jerarquía Visual Intencional: Emplea la escala tipográfica (--font-*-size, --font-*-weight) y los colores (--color-text-*, --color-primary, --color-accent-*) para diferenciar claramente la importancia de la información. Los títulos deben destacar, el texto de cuerpo ser legible, y las acciones primarias (botones --primary-*) ser las más prominentes en su contexto.

Claridad y Legibilidad: Asegura un contraste adecuado entre texto y fondo (especialmente con colores secundarios y placeholders). Utiliza el line-height especificado para facilitar la lectura de párrafos. Etiqueta (<label>) claramente todos los campos de formulario y controles interactivos. Los placeholders deben ser sugerentes, no sustitutos de labels.

Affordance y Feedback: Los elementos interactivos (botones, enlaces, selects, tabs) deben parecer interactivos. Utiliza los estados :hover y :focus-visible de forma consistente para indicar interactividad. Las acciones deben proporcionar feedback claro:

Click: Un estado :active sutil (ej. transform: translateY(1px) o cambio de sombra/fondo ligero).

Carga/Proceso: Deshabilitar (:disabled) el botón de acción y añadir un indicador visual de carga (spinner) si la operación es asíncrona.

Resultado: Confirmación clara mediante Modales (lexy-modal--confirmation), mensajes de éxito/error (idealmente en línea o mediante sistema de notificaciones/toasts - la implementación específica de toasts está fuera de esta guía visual, pero el feedback es necesario).

Accesibilidad (Base):

Utiliza HTML semántico (<button>, <nav>, <main>, <aside>, <header>, <footer>, etc.).

Asocia siempre <label>s con sus <input>s (htmlFor/id).

Proporciona texto alternativo (alt) descriptivo para imágenes significativas.

Asegura que todos los elementos interactivos sean navegables y operables por teclado. El estilo :focus-visible definido es crucial para esto.

Verifica contrastes de color mínimos (WCAG AA).

Contextualidad Funcional:

Aplica los tipos de layout (Ficha vs. SaaS) según el propósito de la pantalla.

Utiliza los Headers y Footers apropiados para cada tipo de layout y flujo. No incluyas un footer si la vista no lo requiere contextualmente.

Elige las variantes de color de botones según la prioridad de la acción en el contexto actual (Primario > Secundario > Contextual > Outline Neutro > Texto/Icono).

Simplicidad y Minimalismo: Evita elementos visuales superfluos. Cada línea, borde, sombra y color debe tener un propósito claro dentro de la jerarquía o la funcionalidad. Sigue el principio de "menos es más" inherente al diseño Lexy.
